# Booksore_project
Bookstore Database Management SystemProject OverviewThis project demonstrates the design and implementation of a relational database for a retail bookstore. It covers the full lifecycle of database management, including schema creation, data seeding, and complex querying for business insights.The system tracks inventory, manages customer profiles, and records sales transactions with referential integrity to ensure data consistency across the platform.Database SchemaThe database consists of four primary tables:BOOKS: Stores catalog information (Title, Author, ISBN, Price, and Stock Levels). CUSTOMERS: Manages user profiles and contact information. SALES: Records the header of each transaction, including total amounts and timestamps. SALE_DETAILS: A junction table for many-to-many relationships between Sales and Books, tracking specific quantities and prices at the time of purchase. Key FeaturesData Integrity: Implements PRIMARY KEY, UNIQUE (for ISBN/Email), and CHECK constraints (to ensure prices are always positive). Automated Updates: Uses ON DELETE CASCADE and ON UPDATE CASCADE on foreign keys to maintain referential integrity automatically. Advanced Logic:Subqueries & Joins: Demonstrates multiple methods for inserting data by retrieving IDs dynamically from other tables. Inventory Management: Scripts for updating stock levels following a sale. Cleaning Operations: Advanced DELETE queries using NOT EXISTS to identify and remove "ghost" customers who haven't made any purchases.
